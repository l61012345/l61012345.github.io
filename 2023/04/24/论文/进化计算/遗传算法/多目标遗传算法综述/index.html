её•зҙҜжүҳжңҖдјҳзҗҶи®ә

еңЁж•°еӯҰдёҠпјҢеӨҡзӣ®ж ҮдјҳеҢ–й—®йўҳеҸҜд»Ҙиў«е…¬ејҸеҢ–дёәеҰӮдёӢиЎЁиҝ°пјҡ
еҰӮжһңеӨҡзӣ®ж Үй—®йўҳзҡ„жҜҸдёҖз§Қи§ЈйғҪеҸҜд»ҘдҪҝз”Ё\(n\)з»ҙз©әй—ҙ\(X\)дёӯзҡ„дёҖдёӘзҡ„еҗ‘йҮҸиҝӣиЎҢиЎЁзӨәпјҡ\(\boldsymbol{x}=\{x_1,x_2,...,x_n\}\)пјҢйӮЈд№ҲеӨҡзӣ®ж ҮдјҳеҢ–е°ұжҳҜжүҫеҲ°дёҖдёӘи§Ј\(\boldsymbol{x^*}\)дҪҝеҫ—\(K\)дёӘзӣ®ж ҮеҮҪж•°(objective function)\(z_i(\boldsymbol{x})\)е…·жңүжңҖе°ҸеҖј:
\[z(x^*)=\{z_1(\boldsymbol{x^*}),z_2(\boldsymbol{x^*}),...,z_n(\boldsymbol{x^*})\}\]

еҗҢж—¶пјҢжҗңзҙўз©әй—ҙ\(X\)еҸ—еҲ°дёҖзі»еҲ—зҡ„зәҰжқҹжқЎд»¶(constrain)\(g\)зҡ„йҷҗеҲ¶пјҡ\(g_j(\boldsymbol{x^*})=b_j\)пјҢ\(j=1,...,m\).

е®һйҷ…жғ…еҶөжҳҜпјҢи®ёеӨҡзәҰжқҹзҡ„ж»Ўи¶іжқЎд»¶еҪјжӯӨд№Ӣй—ҙдјҡеҶІзӘҒпјҢеӣ жӯӨе®һйҷ…дёҠжғіиҰҒжүҫеҲ°дёҖдёӘеӨҡзӣ®ж Үи§ЈдҪҝеҫ—жүҖжңүзӣ®ж ҮеҮҪж•°еҗҢж—¶иҫҫеҲ°жңҖдјҳи§ЈеҮ д№ҺжҳҜдёҚеҸҜиғҪзҡ„гҖӮеӣ жӯӨпјҢдәӢе®һдёҠеӨҡзӣ®ж ҮдјҳеҢ–жҳҜеҜ»жүҫдёҖз»„жңҖдјҳеҢ–зҡ„иҝ‘дјји§ЈпјҢиҝҷдёӘи§ЈеңЁжҜҸдёҖдёӘзӣ®ж ҮеҮҪж•°дёӯзҡ„з»“жһңйғҪеңЁеҸҜд»ҘжҺҘеҸ—зҡ„иҢғеӣҙеҶ…пјҢиҖҢдё”жІЎжңүе…¶д»–зҡ„и§ЈеҸҜд»ҘжҜ”иҝҷдёӘи§ЈиҮіе°‘еңЁдёҖдёӘзӣ®ж ҮеҮҪж•°дёӯжӣҙеҘҪгҖӮ

ж”Ҝй…Қе’Ңеё•зҙҜжүҳжңҖдјҳи§Ј

еҒҮи®ҫдјҳеҢ–й—®йўҳдёәжңҖе°ҸеҖјдјҳеҢ–пјҢеҰӮжһңдёҖдёӘи§Ј\(\boldsymbol{x}\)иҮіе°‘еңЁдёҖдёӘзӣ®ж ҮеҮҪж•°дёӯзҡ„з»“жһңжҜ”еҸҰдёҖдёӘи§Ј\(\boldsymbol{y}\)еҘҪпјҡ\(z_i(\boldsymbol{x})<z_i(\boldsymbol{y}), i=1,2,...,K\)пјҢдё”еңЁе…¶д»–зӣ®ж ҮеҮҪж•°дёӯзҡ„з»“жһңдёҚжҜ”\(\boldsymbol{y}\)е·®пјҡ\(z_j(\boldsymbol{x})вүӨz_j(\boldsymbol{y}),j=1,2,...,K\)пјҢйӮЈд№Ҳз§°и§Ј\(\boldsymbol{x}\)ж”Ҝй…Қ(dominate)еҸҰдёҖдёӘи§Ј\(\boldsymbol{y}\)(\(\boldsymbol{x}вү»\boldsymbol{y}\)).
иҝӣдёҖжӯҘең°пјҢеҰӮжһңдёҖдёӘи§Ј\(\boldsymbol{x}\)еңЁжҗңзҙўз©әй—ҙдёӯдёҚиў«е…¶д»–д»»дҪ•дёҖдёӘи§Јж”Ҝй…ҚпјҢйӮЈд№ҲиҝҷдёӘи§Јиў«з§°дёәеё•зҙҜжүҳжңҖдјҳи§Ј(Pareto optimalпјҢд»ҘдёӢз®Җз§°её•зҙҜжүҳи§Ј)жҲ–иҖ…её•зҙҜжүҳж•ҲзҺҮ(Pareto efficiency)гҖӮ

еңЁжҗңзҙўз©әй—ҙдёӯеӯҳеңЁеӨҡдёӘеё•зҙҜжүҳжңҖдјҳи§ЈпјҢжҜ”еҰӮеҜ№дәҺзӣ®ж ҮеҮҪж•°\(f_1(x),f_2(x)\)пјҢ\([1,2]\)е’Ң\([2,1]\)зӣёдә’д№Ӣй—ҙдёҚиў«ж”Ҝй…ҚгҖӮ

жүҖжңүзҡ„её•зҙҜжүҳжңҖдјҳи§Јжһ„жҲҗзҡ„йӣҶеҗҲз§°дёәеё•зҙҜжүҳжңҖдјҳи§ЈйӣҶеҗҲ(Pareto optimal set)пјҢиҝҷдәӣеё•зҙҜжүҳжңҖдјҳи§ЈеёҰе…ҘеҲ°жҜҸдёҖдёӘзӣ®ж ҮеҮҪж•°дёӯеҫ—еҲ°зҡ„з»“жһңжүҖжһ„жҲҗзҡ„йӣҶеҗҲз§°дёәеё•зҙҜжүҳеүҚжІҝ(PF, Pareto Front).

еӣ жӯӨпјҢеӨҡзӣ®ж ҮдјҳеҢ–зҡ„е…ій”®жҳҜеңЁдәҺжүҫеҲ°её•зҙҜжүҳжңҖдјҳи§ЈйӣҶеҗҲгҖӮ
дҪҶжҳҜпјҢйҡҸзқҖжҗңзҙўз©әй—ҙ\(X\)з»ҙеәҰзҡ„еҚҮй«ҳпјҢеё•зҙҜжүҳйӣҶеҗҲзҡ„ж•°йҮҸдјҡи¶ҠжқҘи¶ҠеӨ§гҖӮеҜ№дәҺеӨ§еӨҡж•°й—®йўҳиҖҢиЁҖпјҢеё•зҙҜжүҳи§Јзҡ„ж•°йҮҸе·ЁеӨ§з”ҡиҮіжҳҜж— з©·еӨҡзҡ„гҖӮе®һйҷ…иҰҒжғіжүҫе®ҢжүҖжңүзҡ„её•зҙҜжүҳжңҖдјҳи§ЈжҳҜдёҚеҸҜиғҪзҡ„гҖӮе№¶дё”иҜҒжҳҺиҝҷдәӣи§Јзҡ„жңҖдјҳжҖ§еңЁи®Ўз®—дёҠд№ҹжҳҜж— жі•е®һзҺ°зҡ„гҖӮеӣ жӯӨе®һйҷ…зҡ„еӨҡзӣ®ж ҮдјҳеҢ–жҳҜжүҫеҲ°жңҖжҺҘиҝ‘е…Ёи§Јзҡ„её•зҙҜжүҳжңҖдјҳи§ЈйӣҶеҗҲгҖӮиҰҒжғійҖјиҝ‘е®Ңж•ҙзҡ„её•зҙҜжүҳжңҖдјҳи§ЈйӣҶеҗҲпјҢжңҖдјҳеҢ–з®—жі•йңҖиҰҒе°ҪеҸҜиғҪең°иҫҫеҲ°дёүдёӘзӣёдә’еҲ¶зәҰзҡ„зӣ®ж Үпјҡ

  1. жүҫеҲ°зҡ„её•зҙҜжүҳеүҚжІҝзҡ„иҝ‘дјјйңҖиҰҒе°ҪеҸҜиғҪең°жҺҘиҝ‘еӨҡзӣ®ж Үй—®йўҳзңҹжӯЈзҡ„её•зҙҜжүҳеүҚжІҝгҖӮ
  2. жүҫеҲ°зҡ„её•зҙҜжүҳиҝ‘дјјжңҖдјҳи§ЈйӣҶеҗҲдёӯпјҢи§ЈеңЁжҗңзҙўз©әй—ҙдёӯзҡ„еҲҶеёғжҳҜе°ҪеҸҜиғҪеқҮеҢҖзҡ„пјҢеҰӮжӯӨиҝ‘дјјжңҖдјҳи§Јзҡ„йӣҶеҗҲдёӯзҡ„и§Је…·жңүдё°еҜҢзҡ„еӨҡж ·жҖ§пјҢдёҚдјҡеҒҸеҗ‘дәҺд»»дҪ•дёҖдёӘзү№е®ҡзҡ„зӣ®ж ҮеҮҪж•°иҖҢжҳҜж•ҙдҪ“ең°иҫҫеҲ°иҝ‘дјјжңҖдјҳгҖӮиҝҷж ·еҸҜд»Ҙи®©еҶізӯ–иҖ…ж— еҒҸең°еңЁиҝҷдәӣеё•зҙҜжүҳи§ЈдёӯжқғиЎЎгҖӮ
  3. жүҫеҲ°зҡ„её•зҙҜжүҳеүҚжІҝзҡ„иҝ‘дјјйңҖиҰҒе°ҪеҸҜиғҪзҡ„жҚ•жҚүзңҹжӯЈеё•зҙҜжүҳеүҚжІҝзҡ„еҲҶеёғзү№жҖ§гҖӮд№ҹе°ұжҳҜиҜҙпјҢжңҖдјҳеҢ–з®—жі•йңҖиҰҒе°ҪеҸҜиғҪзҡ„жүҫеҲ°её•зҙҜжүҳжңҖдјҳи§ЈйӣҶеҗҲдёӯйӮЈдәӣеҸҜиғҪеҲҶеёғеңЁжҗңзҙўз©әй—ҙи§’иҗҪзҡ„жһҒз«Ҝеё•зҙҜжүҳи§ЈгҖӮ

еҜ№дәҺз»ҷе®ҡзҡ„и®Ўз®—е’Ңж—¶й—ҙйҷҗеҲ¶пјҢз¬¬дёҖдёӘзӣ®ж ҮеҸҜд»ҘйҖҡиҝҮе°ҶжҗңзҙўиҒҡз„ҰеңЁеё•зҙҜжүҳеүҚжІҝзҡ„жҹҗдёҖдёӘйғЁеҲҶиҫҫжҲҗпјӣдҪҶжҳҜзӣёеҜ№ең°пјҢз¬¬дәҢдёӘзӣ®ж ҮиҰҒжұӮз®—жі•зҡ„жҗңзҙўйңҖиҰҒеқҮеҢҖж— еҒҸпјӣз¬¬дёүдёӘзӣ®ж ҮиҰҒжұӮз®—жі•е°ҪеҸҜиғҪең°жү©еұ•е·Із»ҸжүҫеҲ°зҡ„её•зҙҜжүҳеүҚжІҝзҡ„дёӨдҫ§пјҢз»§з»ӯеҜ»жүҫж–°зҡ„и§ЈгҖӮ

йҖӮеә”еәҰеҮҪж•°

зәҝжҖ§з»„еҗҲе’ҢиөӢжқғ

жңҖз»Ҹе…ёзҡ„дёҖз§Қи®ҫи®ЎеӨҡзӣ®ж ҮйҒ—дј з®—жі•йҖӮеә”еәҰеҮҪж•°зҡ„ж–№жі•жҳҜе°ҶжүҖжңүзҡ„йҖӮеә”еәҰеҮҪж•°иҝӣиЎҢзәҝжҖ§з»„еҗҲпјҢе№¶еҲҶеҲ«иөӢдәҲжқғйҮҚ\(w_i\)гҖӮеҰӮжӯӨеӨҡзӣ®ж ҮдјҳеҢ–е°ҶйҖҡиҝҮзәҝжҖ§з»„еҗҲиҪ¬еҢ–дёәдёҖз§Қиҝ‘дјјдәҺеҚ•зӣ®ж ҮдјҳеҢ–й—®йўҳзҡ„жұӮи§Јж–№ејҸпјҡ
\[min [z] = w_1z_1(\boldsymbol{x})+w_2z_2(\boldsymbol{x})+...+w_kz_k(\boldsymbol{x})\] йҖҡеёёжүҖжңүзҡ„жқғйҮҚе’Ңдёә1пјҡ\(вҲ‘w_i=1\)гҖӮиҝҷз§Қж–№жі•йҖҡеёёйңҖиҰҒдҪҝз”ЁиҖ…йҖҡиҝҮжҹҗз§Қж–№ејҸз»ҷеҮәжқғйҮҚ\(\boldsymbol{w_i}=\{w_1,w_2,...,w_k\}\)пјҢеёёи§Ғзҡ„иөӢжқғж–№ејҸдёәжҢүз…§дјҳе…Ҳзә§иөӢжқғпјҢеӣ жӯӨиҝҷз§Қж–№жі•иў«з§°дёәдјҳе…Ҳзә§ж–№жі•(priori approach)гҖӮ
еңЁиҝҷз§Қж–№жі•дёӢпјҢз®—жі•йңҖиҰҒдҪҝз”ЁдёҚеҗҢзҡ„жқғйҮҚеӨҡж¬ЎиҝҗиЎҢиҝӣиЎҢжөӢиҜ•пјҢеҰӮдҪ•и®ҫзҪ®дёҚеҗҢзҡ„жқғйҮҚжҳҜиҜҘж–№жі•дёӯзҡ„дёҖдёӘйҮҚиҰҒй—®йўҳгҖӮWBGAдёӯдҪҝз”ЁдәҶдёҖз§ҚиҮӘеҠЁеҢ–иөӢжқғзҡ„ж–№ејҸпјҡWBGAдёӯпјҢжқғйҮҚдёҺдёӘдҪ“зҡ„жҹ“иүІдҪ“з»‘е®ҡпјҢи·ҹйҡҸдёӘдҪ“дёҖеҗҢиў«йҖүжӢ©е’ҢиҝӣеҢ–гҖӮ еҸҰдёҖз§ҚеҹәдәҺMOGAзҡ„еӨҡзӣ®ж ҮйҒ—дј з®—жі•RWGAеҲҷеңЁйҖүжӢ©йҳ¶ж®өдёәжҜҸдёҖдёӘдёӘдҪ“дҪҝз”ЁйҡҸжңәз”ҹжҲҗзҡ„дёҚеҗҢзҡ„жқғйҮҚеҗ‘йҮҸ\(\boldsymbol{w_i}\)гҖӮдҪҝз”ЁйҡҸжңәиөӢжқғзҡ„зӣ®зҡ„жҳҜйҖҡиҝҮйҡҸжңәзҡ„жқғйҮҚеҸҜд»ҘеңЁдёҖж¬Ўз®—жі•иҝҗиЎҢдёӯи§„е®ҡдёҚеҗҢзҡ„жҗңзҙўж–№еҗ‘иҖҢдёҚйңҖиҰҒж·»еҠ е’Ңи®ҫзҪ®е…¶д»–еҸӮж•°гҖӮ

Procedure RWGA:
\(E\): external archive to store non-dominated solutions found during the search so far;
\(n_E\): number of elitist solutions immigrating from \(E\) to \(P\) in each generation.

  • Step 1: Generate a random population.
  • Step 2: Assign a fitness value to each solution \(xвҲҲP_t\) by performing the following steps:
    • Step 2.1: Generate a random number uk in \([0,1]\) for each objective \(k, k=1,...,K\).
    • Step 2.2: Calculate the random weight of each objective \(k\) as \(w_k=(1/u_k)\sum_{i=1}^Ku_i\).
    • Step 2.3: Calculate the fitness of the solution as \(f(\boldsymbol{x})=\sum_{i=1}^Kw_kz_k(\boldsymbol{x})\).
  • Step 3: Calculate the selection probability of each solution \(xвҲҲP_t\) as follows: \(p(\boldsymbol{x})=(f(\boldsymbol{x})-f^{min})^{-1}\sum_{\boldsymbol{y}вҲҲP_t}(f(\boldsymbol{y})-f^{min})\)
    where \(f^{min}=min\{f(\boldsymbol{x})|\boldsymbol{x}вҲҲP_t\}\).
  • Step 4: Select parents using the selection probabilities calculated in Step 3. Apply crossover on the selected parent pairs to create \(N\) offspring. Mutate offspring with a predefined mutation rate.
    Copy all offspring to \(P_{t+1}\).
    Update \(E\) if necessary.
  • Step 5: Randomly remove nE solutions from \(P_{t+1}\) and add the same number of solutions from \(E\) to \(P_{t+1}\).
  • Step 6: If the stopping condition is not satisfied, set \(t=t+1\) and go to Step 2. Otherwise, return to \(E\).

зәҝжҖ§з»„еҗҲзҡ„дё»иҰҒдјҳзӮ№жҳҜиҝҷз§Қж–№жі•йқһеёёз®ҖеҚ•иҖҢдё”жҳ“дәҺе®һзҺ°гҖӮзӣёеҜ№ең°пјҢзәҝжҖ§з»„еҗҲзҡ„ж–№ејҸжҹҘжүҫеҜ№йқһеҮёзҡ„зӣ®ж ҮеҮҪж•°жҖ§иғҪиҫғе·®гҖӮеӣ жӯӨпјҢеҰӮжһңзңҹжӯЈзҡ„её•зҙҜжүҳеүҚжІҝжҳҜйқһеҮёеҮҪж•°пјҢжҢүз…§зәҝжҖ§з»„еҗҲе№¶дёҚиғҪе®Ңе…ЁжүҫеҲ°её•зҙҜжүҳжңҖдјҳи§ЈпјҢиҖҢдё”жүҫеҲ°зҡ„жңҖдјҳи§ЈеҫҲйҡҫеҒҡеҲ°еқҮеҢҖеҲҶеёғгҖӮ

ж”№еҸҳзӣ®ж ҮеҮҪж•°

еңЁжңҖеҲқзҡ„еӨҡзӣ®ж ҮйҒ—дј з®—жі•VEGAдёӯпјҢдҪҝз”ЁдәҶеҸҰдёҖз§ҚйҖӮеә”еәҰеҮҪж•°гҖӮVEGAж №жҚ®зӣ®ж ҮеҮҪж•°зҡ„дёӘж•°е°Ҷж•ҙдёӘз§ҚзҫӨеҲҶжҲҗдәҶиӢҘе№ІдёӘеӯҗз§ҚзҫӨ:\(P_1,P_2,...,P_K\)гҖӮжҜҸдёӘеӯҗз§ҚзҫӨ\(P_i\)еҜ№еә”дёҖдёӘзү№е®ҡзҡ„зӣ®ж ҮеҮҪж•°\(z_i\)пјҢжҜҸдёҖд»ЈдёӯжҢүз…§иҜҘзӣ®ж ҮеҮҪж•°иөӢдәҲйҖӮеә”еәҰгҖҒиҝӣиЎҢйҖүжӢ©еҗҺеҶҚз»јеҗҲиҝӣиЎҢдәӨеҸүе’ҢеҸҳејӮзӯүйҒ—дј ж“ҚдҪңгҖӮ

Procedure VEGA:
\(N_S\): subpopulation size (\(N_S=N/K\)).
- Step 1: Start with a random initial population \(P_0\). Set \(t = 0\).
- Step 2: If the stopping criterion is satisfied, return \(P_t\). - Step 3: Randomly sort population \(P_t\).
- Step 4: For each objective \(k, k= 1,...,K\), perform the following steps:
- Step 4.1: For \(i = 1+(k-1)N_S, ... ,kN_S\), assign fitness value \(f(\boldsymbol{x}_i) = z_k(\boldsymbol{x}_i)\) to the ith solution in the sorted population.
- Step 4.2: Based on the fitness values assigned in Step 4.1, select \(N_S\) solutions between the \((1+(k-1)N_S)\)th and \((kN_S)\)th solutions of the sorted population to create subpopulation \(P_k\).

  • Step 5: Combine all subpopulations \(P_1,...,P_k\) and apply crossover and mutation on the combined population to create \(P_{t+1}\) of size \(N\). Set \(t = t + 1\), go to Step 2.

иҝҷз§Қж–№жі•дё»иҰҒзҡ„дјҳзӮ№жҳҜжҳ“дәҺе®һзҺ°пјҢиҖҢдё”VEGAзҡ„и®Ўз®—ж•ҲзҺҮеҮ д№Һе’ҢеҚ•зӣ®ж ҮйҒ—дј з®—жі•зӣёеҗҢгҖӮдәӢе®һдёҠйҷӨдәҶиҜ„дј°дёҖжӯҘпјҢе…¶дҪҷжӯҘйӘӨVEGAзҡ„иЎҢдёәе’ҢеҚ•зӣ®ж Үдј з»ҹйҒ—дј з®—жі•жҳҜзӣёеҗҢзҡ„гҖӮ
дёҚиҝҮпјҢз§ҚзҫӨеҜ№дәҺдёҚеҗҢзҡ„зӣ®ж ҮеҮҪж•°зҡ„ж•Ҹж„ҹзЁӢеәҰдёҚеҗҢпјҢд№ҹе°ұжҳҜиҜҙпјҢдёҚеҗҢзӣ®ж ҮеҮҪж•°дё»еҜјиҝӣеҢ–ж–№еҗ‘зҡ„иғҪеҠӣжҳҜдёҚеҗҢзҡ„пјҢдё”еңЁVEGAиҝҷж ·зҡ„ж–№жі•дёӯеӯҳеңЁз«һдәүе…ізі»гҖӮеӣ жӯӨз§ҚзҫӨзҡ„иҝӣеҢ–ж–№еҗ‘еҸҜиғҪдјҡеҒҸеҗ‘дәҺе…¶дёӯдёҖдёӘеҜ№е…¶ж•Ҹж„ҹзҡ„зӣ®ж ҮеҮҪж•°пјҢеҜ№е…¶д»–зӣ®ж ҮеҮҪж•°зҡ„ж”¶ж•ӣжҖ§зӣёеҜ№жӣҙе·®гҖӮ

её•зҙҜжүҳжҺ’еәҸ(Pareto ranking)еҲ©з”ЁдәҶеё•зҙҜжүҳжңҖдјҳзҗҶи®әдёӯвҖңж”Ҝй…ҚвҖқзҡ„жҰӮеҝөжқҘдёәдёҚеҗҢзҡ„дёӘдҪ“иөӢдәҲйҖӮеә”еәҰгҖӮз®ҖеҚ•жқҘиҜҙпјҢз§ҚзҫӨдёӯзҡ„дёӘдҪ“йҖӮеә”еәҰиЎЁзӨәдёәжҺ’еәҸгҖӮе…·жңүжӣҙй«ҳж”Ҝй…Қең°дҪҚзҡ„дёӘдҪ“дјҡе…·жңүжӣҙеҘҪзҡ„йҖӮеә”еәҰгҖӮеңЁжң¬ж–ҮдёӯпјҢи¶ҠеҘҪзҡ„дёӘдҪ“е…·жңүжӣҙдҪҺзҡ„йҖӮеә”еәҰпјҢжҺ’еҗҚжӣҙе°Ҹжӣҙйқ еүҚгҖӮ
Goldberg [2] жҸҗеҮәдәҶз¬¬дёҖз§Қеё•зҙҜжүҳжҺ’еәҸж–№жі•пјҢз®ҖеҚ•жқҘиҜҙе°ұжҳҜеҜ№дәҺеҪ“еүҚз§ҚзҫӨдёӯзҡ„жүҖжңүдёӘдҪ“пјҢйҖүжӢ©еҮәе…¶дёӯзҡ„её•зҙҜжүҳи§Је№¶з§»е…ҘеҸҰдёҖдёӘеҲ—иЎЁдёӯпјҢз„¶еҗҺеҶҚзӯӣйҖүеү©дҪҷдёӘдҪ“дёӯзҡ„ж–°зҡ„её•зҙҜжүҳи§ЈгҖӮпјҲеҚіеҺ»жҺүдёҠдёҖж¬Ўзҡ„её•зҙҜжүҳи§ЈеҗҺзҺ°жңүйӣҶеҗҲдёӯж–°зҡ„дёҚеҸ—е…¶д»–з§ҚзҫӨж”Ҝй…Қзҡ„дёӘдҪ“пјүпјҢеҰӮжӯӨеҫҖеӨҚгҖӮ
е…¶иҝҮзЁӢиҜҰз»ҶжҸҸиҝ°еҰӮдёӢпјҡ
- Step 1: Set \(i=1\) and \(TP=P\).
- Step 2: Identify none-dominated solutions in \(TP\) and assigned them set to \(F_i\).
- Step 3: Set \(TP=TPF_i\). If \(TP=вҲ…\) go to Step 4, else set \(i=i+1\) and go to Step 2.
- For every solution \(\boldsymbol{x}вҲҲP\) at generation \(t\), assign \(r(\boldsymbol{x},t)=i\text{ if }\boldsymbol{x}вҲҲF_i\).

еҰӮжӯӨжҺ’еәҸзҡ„з»“жһңжңҖз»Ҳдјҡз”ҹжҲҗиӢҘе№ІдёӘе…·жңүж”Ҝй…ҚеұӮзә§е…ізі»зҡ„жҜҸж¬ЎеҹәдәҺеҪ“еүҚдёӘдҪ“йӣҶеҗҲзҡ„её•зҙҜжүҳеүҚжІҝ\(F_1,F_2,...\)пјҢз§°дёәйқһж”Ҝй…ҚеүҚжІҝ(none-dominated fronts). \(F_1\)жҳҜеҪ“еүҚз§ҚзҫӨдёӯжүҖжңүдёӘдҪ“зҡ„её•зҙҜжүҳеүҚжІҝгҖӮ
е№¶дё”еҸҜд»ҘеҸ‘зҺ°пјҢ\(F_i\)зҡ„ж”Ҝй…ҚзЁӢеәҰиҰҒй«ҳдәҺ\(F_{i+1}\)дёӯзҡ„дёӘдҪ“гҖӮNSGAдҪҝз”ЁдәҶиҝҷз§ҚжҺ’еәҸж–№жі•пјҢе№¶дё”иҝҳжҳҜз”ЁдәҶеҶ—дҪҷзҡ„йҖӮеә”еәҰ(dummy fitness)дҪҝеҫ—\(F_i\)дёӯжңҖе·®дёӘдҪ“зҡ„йҖӮеә”еәҰ/жҺ’еҗҚд»Қз„¶дјҳдәҺ\(F_{i+1}\)дёӯжңҖеҘҪдёӘдҪ“зҡ„йҖӮеә”еәҰ/жҺ’еҗҚгҖӮ
еңЁNSGA-IIдёӯдҪҝз”ЁдәҶжӣҙеҝ«йҖҹзҡ„её•зҙҜжүҳжҺ’еәҸз®—жі•пјҢжҜҸдёӘдёӘдҪ“зҡ„жҺ’еҗҚиЎЁзӨәеҰӮдёӢпјҡ

SPEAеўһж·»дәҶдёҖдёӘеӨ–зҪ®еҲ—иЎЁ\(E\)з”ЁдәҺеӯҳеӮЁеҺҶеҸІдёҠжүҖжңүзҡ„йқһж”Ҝй…ҚдёӘдҪ“гҖӮеҜ№дәҺеңЁ\(E\)дёӯзҡ„жҜҸдёҖдёӘдёӘдҪ“\(\boldsymbol{y}вҲҲE\)пјҢе®ҡд№үпјҡ

\[r(\boldsymbol{x},t)=1+\sum_{\boldsymbol{y}вҲҲE,\boldsymbol{y}вү»\boldsymbol{x}}s(\boldsymbol{y},t)\] иҝҷз§ҚжҺ’еҗҚдёӢпјҢж”Ҝй…Қи¶Ҡе°‘дёӘдҪ“зҡ„йқһж”Ҝй…Қи§Је…·жңүжӣҙдјҳзҡ„йҖӮеә”еәҰпјҢдҪҝеҫ—ж•ҙдёӘеё•зҙҜжүҳеүҚжІҝдёӯзҡ„её•зҙҜжүҳи§Јжӣҙе®№жҳ“еҲҶеёғеқҮеҢҖгҖӮ
жҜ”еҰӮдёӢеӣҫдёӯзҡ„дёӘдҪ“\(a\)е°ұе…·жңүжӣҙдјҳзҡ„йҖӮеә”еәҰпјҢеӣ дёәе®ғеңЁ15дёӘдёӘдҪ“дёӯеҸӘж”Ҝй…ҚдәҶ2дёӘдёӘдҪ“гҖӮиҖҢ\(b\)ж”Ҝй…ҚдәҶ7дёӘдёӘдҪ“пјҢ\(c,d,e\)еҲҶеҲ«ж”Ҝй…ҚдәҶ5пјҢ4пјҢ3дёӘдёӘдҪ“гҖӮ

з§ҜзҙҜжҺ’еәҸеҜҶеәҰзӯ–з•Ҙ(Accumulated Ranking Density Strategy)еҗҢж ·ж—ЁеңЁжғ©зҪҡзЁ еҜҶеҢәзҡ„дёӘдҪ“пјҢе…¶дёӯзҡ„жҺ’еҗҚиЎЁзӨәдёәпјҡ
\[r(\boldsymbol{x},t)=1+\sum_{\boldsymbol{y}вҲҲP,\boldsymbol{y}вү»\boldsymbol{x}}r(\boldsymbol{y},t)\] иҝҷз§ҚжҺ’еҗҚйҒҝе…ҚдәҶдҪҝз”ЁеӨ–зҪ®еҲ—иЎЁ\(E\)гҖӮдҪҶжҳҜпјҢе…¶жҺ’еҗҚи®Ўз®—еӯҳеңЁе…ҲеҗҺйЎәеәҸпјҡиҰҒжғіи®Ўз®—\(\boldsymbol{x}\)зҡ„жҺ’еҗҚпјҢе°ұеҝ…йЎ»е…ҲжұӮеҮәж”Ҝй…Қ\(\boldsymbol{x}\)зҡ„е…ЁйғЁдёӘдҪ“\(\boldsymbol{y}\)зҡ„жҺ’еҗҚгҖӮ
еңЁиҝҷз§ҚжҺ’еҗҚдёӢпјҢеҸ—еҲ°д№ӢеүҚйқһж”Ҝй…ҚеүҚжІҝдёӯзЁ еҜҶдёӘдҪ“ж”Ҝй…Қзҡ„дёӘдҪ“еңЁжң¬йқһж”Ҝй…ҚеүҚжІҝдёӯе…·жңүжӣҙе·®зҡ„йҖӮеә”еәҰгҖӮжҜ”еҰӮдёӢеӣҫдёӯеҸ—еҲ°\(F_1\)дёӯдёӘдҪ“\(c,d,e\)ж”Ҝй…Қзҡ„дёӘдҪ“\(i,l,n\)еңЁе®ғд»¬еҗ„иҮӘзҡ„йқһж”Ҝй…ҚеүҚжІҝдёӯйғҪе…·жңүжӣҙдҪҺзҡ„жҺ’еҗҚгҖӮ

жҖ»иҖҢиЁҖд№ӢпјҢдёҠиҝ°зҡ„жҺ’еәҸзӯ–з•Ҙеҫ—еҲ°зҡ„жҺ’еҗҚйғҪеҸҜд»ҘзӣҙжҺҘдҪңдёәдёӘдҪ“зҡ„йҖӮеә”еәҰиҝӣиЎҢдҪҝз”ЁгҖӮе®ғд»¬йҖҡеёёз»“еҗҲдёӢйқўжҸҗеҲ°зҡ„йҖӮеә”еәҰе…ұдә«(fitenss sharing)ж–№жі•жқҘеҫ—еҲ°жӣҙдёәе№ҝжіӣе’ҢеқҮеҢҖзҡ„её•зҙҜжүҳеүҚжІҝиҝ‘дјј пјҲж»Ўи¶ізӣ®ж Ү2е’Ң3пјүгҖӮ

еӨҡж ·жҖ§пјҡйҖӮеә”еәҰе…ұдә«е’Ңе°Ҹз”ҹеўғ

дёәдәҶиҰҒеҫ—еҲ°еқҮеҢҖеҲҶеёғзҡ„её•зҙҜжүҳеүҚжІҝиҝ‘дјјпјҢе§Ӣз»ҲдҝқиҜҒеҪ“еүҚз§ҚзҫӨдёӯзҡ„и§ЈиҰҒе…·жңүеӨҡж ·жҖ§пјҢеҚіж¶өзӣ–её•зҙҜжүҳеүҚжІҝзҡ„дёҚеҗҢеҢәеҹҹгҖӮеҰӮжһңжІЎжңүйҮҮеҸ–дҝқжҠӨз§ҚзҫӨеӨҡж ·жҖ§зҡ„ж–№жі•пјҢз§ҚзҫӨжҖ»жҳҜеҖҫеҗ‘дәҺеҪўжҲҗиӢҘе№ІдёӘеҶ…йғЁдёӘдҪ“еҪјжӯӨзӣёдјјзҡ„еӯҗз§ҚзҫӨпјҢиҝҷз§ҚзҺ°иұЎжҲҗдёәеҹәеӣ жјӮеҸҳ(Genetic Drift)гҖӮиҝҷдәӣеӯҗз§ҚзҫӨиў«з§°дёәе°Ҹз”ҹеўғ(niche)гҖӮжңүеҰӮдёӢзҡ„ж–№ејҸеҸҜд»Ҙи§ЈеҶіеҹәеӣ жјӮеҸҳзҡ„й—®йўҳгҖӮ

йҖӮеә”еәҰе…ұдә«

йҖӮеә”еәҰе…ұдә«(fitness sharing)зҡ„зӣ®зҡ„жҳҜдёәдәҶжүҫеҲ°жІЎжңүжҺўзҙўиҝҮзҡ„её•зҙҜжүҳеүҚжІҝзҡ„йғЁеҲҶпјҢеҗҢж—¶еҮҸе°‘её•зҙҜжүҳеүҚжІҝзЁ еҜҶеҢәпјҲdense area, жҢҮдёӘдҪ“еҲҶеёғйқһеёёеҜҶйӣҶзҡ„еҢәеҹҹпјүдёӘдҪ“зҡ„йҖӮеә”еәҰпјҢе®ғйҖҡеёёйҖҡиҝҮи®ҫзҪ®жғ©зҪҡеҮҪж•°(penalty function)жқҘйҷҚдҪҺпјҲжҲ–иҖ…з§°дёәжғ©зҪҡпјүйӮЈдәӣдҪҚдәҺзЁ еҜҶеҢәдёӘдҪ“зҡ„йҖӮеә”еәҰгҖӮиҰҒеҒҡеҲ°иҝҷдёҖзӮ№пјҢе°ұеҝ…йЎ»иҰҒз»ҷеҮәжҳҺзЎ®зҡ„еҜ№дәҺзЁ еҜҶзҡ„е®ҡд№үгҖӮ
Fonsecaе’ҢFlemingдҪҝз”ЁдәҶдёҖз§Қзұ»дјјдәҺеҪ’дёҖеҢ–зҡ„ж¬§ж°Ҹи·қзҰ»жқҘиЎЎйҮҸдёӘдҪ“д№Ӣй—ҙзҡ„и·қзҰ»пјҢе№¶ж №жҚ®жӯӨжқҘз»ҷеҮәеҜ№зЁ еҜҶзҡ„е®ҡд№үгҖӮеңЁд»–д»¬жҸҗеҮәзҡ„ж–№жі•дёӯпјҢзҺ°жңүз§ҚзҫӨдёӯзҡ„жҜҸдёӘдёӘдҪ“йғҪдёҺе…¶д»–жүҖжңүдёӘдҪ“дёӨдёӨй…ҚеҜ№дёҖж¬ЎпјҢеңЁжҜҸдёӘй…ҚеҜ№дёӯпјҢдёӘдҪ“\(\boldsymbol{x}\)е’Ң\(\boldsymbol{y}\)зҡ„и·қзҰ»иЎЁзӨәдёәпјҡ
\[dz(\boldsymbol{x},\boldsymbol{y})=\sqrt{\sum_{k=1}^K\left(\frac{z_k(\boldsymbol{x})-z_k(\boldsymbol{y})}{z_k^{max}-z_k^{min}}\right)^2}\] е…¶дёӯ\(z_k^{max}\)е’Ң\(z_k^{min}\)еҲҶеҲ«жҳҜиҝ„д»Ҡдёәжӯўиў«жҺўзҙўеҲ°зҡ„зӣ®ж ҮеҮҪж•°\(z_k(В·)\)зҡ„жңҖеӨ§еҖје’ҢжңҖе°ҸеҖјгҖӮ
еҜ№дёӘдҪ“\(\boldsymbol{x}\)е…¶йҖӮеә”еәҰ\(f(\boldsymbol{x},t)\)дјҡиў«йҷ„иҝ‘зҡ„дёӘдҪ“зЁҖйҮҠпјҡ
\[f'(\boldsymbol{x},t)=\frac{f(\boldsymbol{x},t)}{nc(\boldsymbol{x},t)}\] е…¶дёӯ\(nc(\boldsymbol{x},t)\)иў«з§°дёә\(\boldsymbol{x}\)зҡ„е°Ҹз”ҹеўғи·қзҰ»(niche count)пјҡ
\[nc(\boldsymbol{x},t)=\sum_{\boldsymbol{y}вҲҲP,r(\boldsymbol{y},t)=r(\boldsymbol{x},t)}max\left\{\frac{Пғ_{share}-dz(\boldsymbol{x},\boldsymbol{y})}{Пғ_{share}},0\right\}\] е…¶дёӯ\(Пғ_{share}\)жҳҜдёҖдёӘйў„е…ҲеҶіе®ҡзҡ„еҸӮж•°пјҢз§°дёәе°Ҹз”ҹеўғеӨ§е°ҸпјҢеҶіе®ҡдәҶйңҖиҰҒиҖғиҷ‘зҡ„дёӘдҪ“иҢғеӣҙгҖӮ еҰӮжӯӨпјҢ\(nc(\boldsymbol{x},t)\)и¶ҠеӨ§пјҢ\(\boldsymbol{x}\)йҷ„иҝ‘и¶ҠзЁ еҜҶпјҢ\(f'(\boldsymbol{x},t)\)и¶Ҡе°ҸгҖӮ

еӣ жӯӨпјҢе°Ҹз”ҹеўғйҷҗеҲ¶дәҶжҹҘжүҫдёӘдҪ“ж•°еңЁжҹҗдёӘзү№е®ҡзҡ„её•зҙҜжүҳеүҚжІҝеҢәеҹҹзҡ„жҝҖеўһгҖӮ
еҸҰдёҖз§Қжӣҝд»Јзҡ„и·қзҰ»еәҰйҮҸжҳҜпјҡ
\[dx(\boldsymbol{x},\boldsymbol{y})=\sqrt{\frac{1}{M}\sum_{i=1}^M(x_i-y_i)^2}\]

и®ёеӨҡз ”з©¶йғҪи§ӮеҜҹеҲ°дҪҝз”ЁйҖӮеә”еәҰе…ұдә«зҡ„еӨҡзӣ®ж ҮдјҳеҢ–зӯ–з•ҘеҫҖеҫҖиҰҒжҜ”дёҚдҪҝз”ЁйҖӮеә”еәҰе…ұдә«зҡ„еӨҡзӣ®ж ҮдјҳеҢ–жӣҙеҘҪгҖӮ

дҪҶжҳҜпјҢйҖӮеә”еәҰе…ұдә«зҡ„е…¶дёӯдёҖдёӘй—®йўҳжҳҜеңЁжҹҘжүҫеүҚйңҖиҰҒзЎ®е®ҡе°Ҹз”ҹеўғеӨ§е°Ҹ\(Пғ_{share}\)гҖӮFonsecaе’ҢFlemin[3] жҸҗеҮәдәҶдёҖз§ҚеҠЁжҖҒи°ғж•ҙ\(Пғ_{share}\)зҡ„ж–№жі•гҖӮеҸҰдёҖдёӘй—®йўҳжҳҜеј•е…Ҙ\(Пғ_{share}\)жүҖеўһеҠ зҡ„и®Ўз®—йҮҸгҖӮ

MOGAжҳҜз¬¬дёҖдёӘеҗҢж—¶дҪҝз”Ёеё•зҙҜжүҳжҺ’еәҸе’Ңе°Ҹз”ҹеўғзҡ„еӨҡзӣ®ж ҮдјҳеҢ–з®—жі•гҖӮ

  • Step 1: Start with a random initial population \(P_0\). Set \(t = 0\).
  • Step 2: If the stopping criterion is satisfied, return \(P_t\).
  • Step 3: Evaluate fitness of the population as follows:
    • Step 3.1: Assign a rank \(r(x,t)\) to each solution \(\boldsymbol{x}вҲҲP_t\) using the ranking scheme: \(r(\boldsymbol{x},t)=1+nq(\boldsymbol{x},t)\).
    • Step 3.2: Assign a fitness values to each solution based on the solutionвҖҷs rank as follows: \[f(\boldsymbol{x},t)=N-\sum_{k=1}^{r(\boldsymbol{x},t)}n_k-0.5(n_{r(\boldsymbol{x},t)}-1)\] е…¶дёӯ\(n_k\)жҳҜе…·жңүжҺ’еҗҚ\(k\)зҡ„дёӘдҪ“зҡ„ж•°зӣ®гҖӮ
    • Step 3.3: Calculate the niche count \(nc(\boldsymbol{x}, t)\) of each solution \(\boldsymbol{x}вҲҲP_t\).
    • Step 3.4: Calculate the shared fitness value of each solution \(\boldsymbol{x}вҲҲP_t\) as follows:
      \[f'(\boldsymbol{x},t)=f(\boldsymbol{x},t)/nc(\boldsymbol{x},t)\]
    • Step 3.5: Normalize the fitness values by using the shared fitness values:
      \[f^{''}(\boldsymbol{x},t)=\frac{f^{'}(\boldsymbol{x},t)n_r(\boldsymbol{x},t)}{вҲ‘_{\boldsymbol{y}вҲҲP_t,r(\boldsymbol{y},t)=r(\boldsymbol{x},t)}f'(\boldsymbol{x},t)}f(\boldsymbol{x},t)\]
  • Step 4: Use a stochastic selection method based on \(f^{''}\) to select parents for the mating pool. Apply crossover and mutation on the mating pool until offspring population \(Q_t\) of size \(N\) is filled. Set \(P_{t+1} = Q_t\).
  • Step 5: Set \(t =t +1\), go to Step 2.

SPEA2дёӯдҪҝз”ЁдәҶдёҖз§Қж–№жі•жқҘеҢәеҲҶе…·жңүзӣёеҗҢжҺ’еҗҚзҡ„дёӘдҪ“пјҡдёҖдёӘдёӘдҪ“е‘Ёеӣҙзҡ„еҜҶеәҰиў«е®ҡд№үдёәдёҺе…¶жңҖиҝ‘зҡ„з¬¬\(k\)дёӘдёӘдҪ“зҡ„и·қзҰ»зҡ„зӣёеҸҚж•°гҖӮи®ҫзҪ®\(k\)жҜ”и®ҫзҪ®е°Ҹз”ҹеўғеӨ§е°Ҹ\(Пғ_{share}\)жӣҙдёәзӣҙжҺҘгҖӮ

жӢҘжҢӨи·қзҰ»

жӢҘжҢӨи·қзҰ»(crowding distance)ж—ЁеңЁдёҚдҪҝз”ЁйҖӮеә”еәҰе…ұдә«ж–№жі•зҡ„еҸӮж•°е°ұеҸҜд»Ҙд»ҺжңҖдјҳзҡ„её•зҙҜжүҳеүҚжІҝиҝ‘дјјдёӯеҫ—еҲ°еқҮеҢҖзҡ„дёӘдҪ“еҲҶеёғгҖӮNSGA-IIдёӯдҪҝз”ЁдәҶдёҖз§ҚжӢҘжҢӨи·қзҰ»зҡ„ж–№жі•пјҡ
еҜ№дәҺдҪҝз”Ёеё•зҙҜжүҳжҺ’еәҸз”ҹжҲҗзҡ„иӢҘе№ІдёӘйқһж”Ҝй…ҚеүҚжІҝ\(F_1,...,F_R\)пјҢеҜ№дәҺжҜҸдёҖдёӘзӣ®ж ҮеҮҪж•°\(k\)пјҢеҜ№\(F_j\)дёӯзҡ„жҜҸдёҖдёӘи§ЈеҚҮеәҸжҺ’еәҸгҖӮи®ҫ\(l=|F_j|\)пјҢ\(\boldsymbol{x}_{[i,k]}\)иЎЁзӨәеҜ№\(k\)дёӘзӣ®ж ҮеҮҪж•°жҺ’еәҸеҲ—иЎЁдёӯзҡ„з¬¬\(i\)дёӘдёӘдҪ“гҖӮд»Ө\(cd_k(\boldsymbol{x}_{[1,k]})=вҲһ\), \(cd_k(\boldsymbol{x}_{[l,k]})=вҲһ\)пјҢеҜ№еү©дёӢзҡ„\(i=2,3,...,l-1\):
\[cd_k(\boldsymbol{x}_{[i,k]})=\frac{z_k(\boldsymbol{x}_{[i+1,k]}-\boldsymbol{x}_{[i-1,k]})}{z_k^{max}-z_k^{min}}\]

еҸҜд»ҘеҸ‘зҺ°пјҢ\(cd\)и¶ҠеӨ§пјҢзӣёйӮ»дёӘдҪ“д№Ӣй—ҙзҡ„й—ҙи·қе°ұи¶ҠеӨ§гҖӮ
жңҖеҗҺпјҢеҜ№жҜҸдёӘ\(\boldsymbol{x}\),еҜ№жүҖжңүзӣ®ж ҮеҮҪж•°зҡ„\(cd_k(\boldsymbol{x})\)е…ЁйғЁжұӮе’Ң:
\[cd(\boldsymbol{x})=вҲ‘_kcd_k(\boldsymbol{x})\]

NSGA-IIдёӯдҪҝз”ЁжӢҘжҢӨи·қзҰ»дҪңдёәзЁ еҜҶеәҰеҸӮиҖғжқҘиҝӣиЎҢйҖүжӢ©пјҢз§°дёәжӢҘжҢӨй”Ұж ҮиөӣйҖүжӢ©(crowed tournament selection). еңЁжӢҘжҢӨй”Ұж ҮиөӣйҖүжӢ©дёӯпјҢйҡҸжңәй…ҚеҜ№зҡ„дёӨдёӘдёӘдҪ“\(\boldsymbol{x}\),\(\boldsymbol{y}\)гҖӮеҰӮжһңдёӨдёӘдёӘдҪ“дҪҚдәҺеҗҢж ·зҡ„йқһж”Ҝй…ҚеүҚжІҝпјҢжңүжӣҙй«ҳ\(cd\)зҡ„дёӘдҪ“е°Ҷиў«йҖүжӢ©пјӣеҰӮжһңдёӨдёӘдёӘдҪ“еңЁдёҚеҗҢзҡ„йқһж”Ҝй…ҚеүҚжІҝпјҢе…·жңүжӣҙдҪҺжҺ’еҗҚзҡ„дёӘдҪ“е°Ҷиў«йҖүжӢ©гҖӮ

дҪҝз”ЁжӢҘжҢӨи·қзҰ»еҘҪеӨ„жҳҜиҝҷз§ҚиЎЎйҮҸдёӘдҪ“зЁ еҜҶеәҰзҡ„ж–№жі•дёҚйңҖиҰҒдҪҝз”ЁиҖ…жҸҗеүҚи®ҫзҪ®еҸӮж•°\(Пғ_{share}\)жҲ–иҖ…жҳҜ\(k\).

еҹәдәҺзҪ‘ж јзҡ„зЁ еҜҶеәҰ

еңЁеҹәдәҺзҪ‘ж јзҡ„зЁ еҜҶеәҰ(cell-based density)ж–№жі•дёӯпјҢжҗңзҙўз©әй—ҙиў«еҲ’еҲҶдёәиӢҘе№ІдёӘ\(K\)з»ҙзҡ„и¶…з«Ӣж–№зҪ‘ж јгҖӮжҜҸдёӘзҪ‘ж јдёӯдёӘдҪ“зҡ„ж•°йҮҸз§°дёәиҜҘзҪ‘ж јзҡ„еҜҶеәҰпјҢе№¶дё”е®ҡд№үдёҖдёӘдёӘдҪ“зҡ„еҜҶеәҰе°ұжҳҜиҜҘдёӘдҪ“жүҖеңЁзҪ‘ж јзҡ„еҜҶеәҰгҖӮиҝҷж ·зҡ„зЁ еҜҶеәҰдҝЎжҒҜеҸҜд»Ҙиҫ…еҠ©е®һзҺ°йҖӮеә”еәҰе…ұдә«гҖӮ
жҜ”еҰӮеңЁPESAдёӯпјҢе…·жңүжӣҙдҪҺзЁ еҜҶеәҰзҡ„дёӘдҪ“жӣҙеҸ—ж¬ўиҝҺгҖӮ

  • Step 1: Start with a random initial population P0 and set external achieve \(E_0 = вҲ…, t = 0\).
  • Step 2: Divide the normalized objective space into \(n^K\) hyper-cubes where \(n\) is the number of grids along a single objective axis and \(K\) is the number of objectives.
  • Step 3: Update non-dominated archive \(E_t\) by incorporating new solutions from \(P_t\) one by one as follows:
    • Case 1: If a new solution is dominated by at least a solution in \(E_t\), discard the new solution.
    • Case 2: If a new solution dominates some solutions in \(E_t\), remove those dominated solutions from \(E_t\) and add to the new solution to \(E_t\). Update the membership of the hyper-cubes.
    • Case 3: If a new solution is not dominated by and does not dominate any solution in \(E_t\), add this solution to \(E_t\). If \(|E_t|=N_E+1\), randomly choose a solution from the most crowded hyper-cubes to be removed. Update the membership of the hyper-cubes.
  • Step 4: If the stopping criterion is satisfied, stop and return \(E_t\).
  • Step 5: Set \(P_t = вҲ…\), and select solutions from \(E_t\) for crossover and mutation based on the density information of the hyper-cubes. For example, if binary tournament selection is used, the winner is the solution located in the less crowded hyper-cubes. Apply crossover and mutation to generate \(N_P\) offspring and copy them to \(P_{t+1}\).
  • Step 6: Set \(t = t + 1\) and go to Step 3.

Procedure NSGA-II:
- Step 1: Create a random parent population \(P_0\) of size \(N\). Set \(t =0\).
- Step 2: Apply crossover and mutation to \(P_0\) to create offspring population \(Q_0\) of size \(N\).
- Step 3: If the stopping criterion is satisfied, stop and return to \(P_t\).
- Step 4: Set \(R_t =P_t вҲӘ Q_t\).
- Step 5: Using the fast non-dominated sorting algorithm, identify the non-dominated fronts \(F_1, F_2, ...,F_k\) in \(R_t\).
- Step 6: For \(i =1,...,k\) do following steps:
- Step 6.1: Calculate crowding distance of the solutions in \(F_i\).
- Step 6.2: Create \(P_{t+1}\) as follows:
- Case 1: If \(| P_{t+1} | + | F_i | вүӨ N\), then set \(P_{t+1} =P_{t+1}вҲӘF_i\); - Case 2: If \(| P_{t+1} | + | F_i | >N\), then add the least crowded \(N- | P_{t+1} |\) solutions from \(F_i\) to \(P_{t+1}\).
- Step 7: Use binary tournament selection based on the crowding distance to select parents from \(P_{t+1}\). Apply crossover and mutation to \(P_{t+1}\) to create offspring population \(Q_{t+1}\) of size \(N\). - Step 8: Set \(t =t + 1\), and go to Step 3.

    • Step 2.2: Calculate the density as: \[m(\boldsymbol{x}, t)=(Пғ^k_{\boldsymbol{x}} + 1)^{-1}\]
      where \(Пғ^k_{\boldsymbol{x}}\) is the distance between solution x and its \(k\) th nearest neighbor.
    • Step 2.3: Assign a fitness value as: \[f (\boldsymbol{x},t) =r(\boldsymbol{x},t) + m(\boldsymbol{x}, t)\]
  • Step 3: Copy all non-dominated solutions in \(P_t вҲӘ E_t\) to \(E_{t+1}\). Two cases are possible:
    • Case 1: If \(| E_{t+1} | >N_E\), then truncate \((| E_{t+1} | -N_E)\) solutions by iteratively removing solutions with the maximum \(\sigma ^k\) distances. Break any tie by examining \(\sigma ^l\) for \(l=k-1,...,1\) sequentially.
    • Case 2: If \(| E_{t+1} | вүӨN_E\), copy the best \((N_E- | E_{t+1} |)\) dominated solutions according to their fitness values from \(P_t вҲӘ E_t\) to \(E_{t+1}\).
  • Step 4: If the stopping criterion is satisfied, stop and return non-dominated solutions in \(E_{t+1}\).
  • Step 5: Select parents from EtГҫ1 using binary tournament selection with replacement.
  • Step 6: Apply crossover and mutation operators to the parents to create NP offspring solutions. Copy offspring to \(P_{t+1}\), \(t = t + 1\), and go to Step 2.

жқЎд»¶йҷҗеҲ¶

еӨ§еӨҡж•°зңҹе®һзҡ„еӨҡзӣ®ж ҮдјҳеҢ–й—®йўҳдёӯпјҢдјҳеҢ–зӣ®ж Үеёёеёёжңүеҗ„з§Қеҗ„ж ·зҡ„йҷҗеҲ¶(constraint)гҖӮеңЁеҚ•зӣ®ж ҮйҒ—дј з®—жі•дёӯпјҢз”ЁжқҘиҝӣиЎҢжқЎд»¶йҷҗеҲ¶зҡ„зӯ–з•Ҙжңүеӣӣз§Қпјҡ
- I. иҲҚејғдёҚеҸҜиғҪе®һзҺ°зҡ„и§ЈеҶіж–№жЎҲпјҲеҸҲз§°дёәвҖңDeath PenaltyвҖқпјү
- II. дҪҝз”Ёжғ©зҪҡеҮҪж•°йҷҚдҪҺдёҚеҸҜиғҪе®һзҺ°дёӘдҪ“зҡ„йҖӮеә”еәҰ
- III. и®ҫи®ЎйҒ—дј ж“ҚдҪңдҪҝеҫ—з§ҚзҫӨжҖ»жҳҜеҸҜд»Ҙдә§з”ҹеҸҜиЎҢзҡ„дёӘдҪ“ - IV. и®ҫи®Ўжҹҗз§ҚжңәеҲ¶дҪҝеҫ—дёҚеҸҜиғҪдёӘдҪ“еҸҳжҲҗеҸҜиЎҢзҡ„дёӘдҪ“пјҲеҸҲз§°дёәвҖңrepairвҖқпјү

зӣ®еүҚпјҢеҜ№еӨҡзӣ®ж ҮйҒ—дј з®—жі•дёӯжқЎд»¶йҷҗеҲ¶й—®йўҳзҡ„з ”з©¶е№¶дёҚе……еҲҶпјҢеӨ§еӨҡж•°зҡ„еӨҡзӣ®ж ҮйҒ—дј з®—жі•е№¶дёҚиҖғиҷ‘жқЎд»¶йҷҗеҲ¶гҖӮдёҚиҝҮпјҢзӯ–з•ҘI, III,IVеҸҜд»ҘзӣҙжҺҘеә”з”ЁдәҺеӨҡзӣ®ж ҮйҒ—дј з®—жі•гҖӮ
Jimenezзӯүдәә[5]жҸҗеҮәдәҶдёҖз§ҚеҹәдәҺе°Ҹз”ҹеўғзҡ„йҖүжӢ©зӯ–з•ҘжқҘе®һзҺ°жқЎд»¶йҷҗеҲ¶пјҢиҝҮзЁӢеҰӮдёӢпјҡ
- Step 1: Randomly chose two solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\) from the population.
- Step 2: If one of the solutions is feasible and the other one is infeasible, the winner is the feasible solution, and stop. Otherwise, if both solutions are infeasible go to Step 3, else go to Step 4.
- Step 3: In this case, solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\) are both infeasible.
Then, select a random reference set \(C\) among infeasible solutions in the population. Compare solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\) to the solutions in reference set \(C\) with respect to their degree of infeasibility. In order to achieve this, calculate a measure of infeasibility (e.g., the number of constraints violated or total constraint violation) for solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\), and those in set \(C\). If one of solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\) is better and the other one is worse than the best solution in \(C\), with respect to the calculated infeasibility measure, then the winner is the least infeasible solution.
However, if there is a tie, that is, both solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\) are either better or worse than the best solution in \(C\), then their niche count in decision variable space is used for selection. In this case, the solution with the lower niche count is the winner.
- Step 4: In the case that solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\) are both feasible, select a random reference set \(C\) among feasible solutions in the population. Compare solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\) to the solutions in set \(C\). If one of them is non-dominated in set \(C\), and the other is dominated by at least one solution, the winner is the former. Otherwise, there is a tie between solutions \(\boldsymbol{x}\) and \(\boldsymbol{y}\), and the niche count of the solutions is calculated in decision variable space. The solution with the smaller niche count is the winner of the tournament selection.

з®ҖеҚ•жқҘиҜҙпјҢиҜҘж–№жі•дҪҝз”ЁдәҶдёҖдёӘеҸӮиҖғйӣҶеҗҲ\(C\)пјҢиҜҘеҸӮиҖғйӣҶеҗҲдёӯзҡ„жүҖжңүдёӘдҪ“йғҪжҳҜеҸҜиЎҢзҡ„гҖӮз„¶еҗҺйҡҸжңәй…ҚеҜ№з§ҚзҫӨдёӯзҡ„дёӨдёӘдёӘдҪ“\(\boldsymbol{x}\) е’Ң \(\boldsymbol{y}\) д»Ҙ \(C\)дёәеҸӮз…§еҜ№жҜ”еҸҜиЎҢжҖ§пјҢеҸҜиЎҢжҖ§жӣҙй«ҳзҡ„дёӘдҪ“иғңеҮәгҖӮеҰӮжһңиҝҷдёӨдёӘдёӘдҪ“жң¬иә«йғҪжҳҜеҸҜиЎҢзҡ„пјҢеҲҷжҜ”иҫғдёӨиҖ…зҡ„е°Ҹз”ҹеўғи·қзҰ»пјҢе°Ҹз”ҹеўғи·қзҰ»жӣҙе°Ҹзҡ„дёӘдҪ“иғңеҮәгҖӮ

дёҠиҝ°зҡ„иҝҷз§Қж–№жі•жҳҜдёҖз§Қиҫғдёәз»јеҗҲзҡ„еӨ„зҗҶйҷҗеҲ¶жқЎд»¶зҡ„ж–№жі•пјҢеңЁеӨ„зҗҶиҝҮзЁӢдёӯиҝҳеҸҜд»Ҙжңүж•Ҳең°дҝқиҜҒдёӘдҪ“еӨҡж ·жҖ§гҖӮиҝҷз§Қж–№жі•зҡ„зјәзӮ№жҳҜи®Ўз®—еӨҚжқӮеәҰд»ҘеҸҠеј•е…ҘдәҶдҫӢеҰӮе°Ҹз”ҹеўғи·қзҰ»е’ҢеҸӮиҖғз§ҚзҫӨ\(C\)зӣёе…ізҡ„и¶…еҸӮж•°гҖӮ

дёҚиҝҮдёҠиҝ°ж–№жі•д»Қз„¶и®ёеӨҡеҸҜд»Ҙж”№иҝӣзҡ„ең°ж–№пјҢжҜ”еҰӮз¬¬дёүжӯҘдёӯ\(\boldsymbol{x}\) е’Ң \(\boldsymbol{y}\)зҡ„еҸҜиЎҢжҖ§еҸҜд»ҘзӣҙжҺҘжҜ”иҫғиҖҢдёҚйңҖиҰҒе®һз”ЁдёҖдёӘеҸӮиҖғйӣҶеҗҲ\(C\);еҶҚжҜ”еҰӮз¬¬еӣӣжӯҘдёӯе°Ҹз”ҹеўғи·қзҰ»еҸҜд»ҘеңЁзӣ®ж ҮеҮҪж•°зҡ„з©әй—ҙдёӯзӣҙжҺҘеҫ—еҮәпјҢдёҚйңҖиҰҒеӣһеҲ°еҸҳйҮҸзҡ„з©әй—ҙ(decision variable space)дёӯеҶҚиҝӣиЎҢи®Ўз®—гҖӮ

DebзӯүдәәжҸҗеҮәзҡ„NSGA-IIдёӯ[6]жҸҗеҮәдәҶжңүйҷҗеҲ¶зҡ„й”Ұж ҮиөӣйҖүжӢ©жңәеҲ¶(constrained tournament selection)гҖӮеңЁиҝҷз§ҚжңәеҲ¶дёӯпјҢеҰӮжһңдёӘдҪ“\(\boldsymbol{x}\)ж»Ўи¶іеҰӮдёӢд»»дҪ•дёҖдёӘжқЎд»¶пјҢз§°дёӘдҪ“\(\boldsymbol{x}\)жңүжқЎд»¶ең°ж”Ҝй…Қ(constrain-dominate)дёӘдҪ“\(\boldsymbol{y}\)пјҡ
- \(\boldsymbol{x}\)еҸҜиЎҢпјҢ\(\boldsymbol{y}\)дёҚеҸҜиЎҢ - \(\boldsymbol{x}\)е’Ң\(\boldsymbol{y}\)йғҪдёҚеҸҜиЎҢпјҢдҪҶжҳҜ\(\boldsymbol{x}\)еҜ№жқЎд»¶зҡ„з ҙеқҸдҪҺдәҺ\(\boldsymbol{y}\) - \(\boldsymbol{x}\)е’Ң\(\boldsymbol{y}\)йғҪеҸҜиЎҢпјҢ\(\boldsymbol{x}\)ж”Ҝй…Қ\(\boldsymbol{y}\)

йҰ–е…ҲпјҢиӢҘе№ІдёӘжқЎд»¶йҷҗеҲ¶зҡ„йқһж”Ҝй…ҚеүҚжІҝ\(F_1,...,F_R\)еҸҜд»ҘйҖҡиҝҮдҪҝз”ЁдёҠиҝ°зҡ„вҖңжңүжқЎд»¶ж”Ҝй…ҚвҖқзҡ„е®ҡд№үиҝӣиЎҢеё•зҙҜжүҳжҺ’еәҸеҫ—еҮәгҖӮеңЁиҝҷж ·зҡ„йҖүжӢ©жңәеҲ¶дёӯпјҢйҡҸжңәйҖүжӢ©дёӨдёӘдёӘдҪ“\(\boldsymbol{x}\) е’Ң \(\boldsymbol{y}\)пјҢз„¶еҗҺжҜ”иҫғдёӨиҖ…еңЁжқЎд»¶йҷҗеҲ¶зҡ„йқһж”Ҝй…ҚеүҚжІҝдёӯзҡ„ең°дҪҚгҖӮ
иҝҷз§Қж–№ејҸзҡ„дјҳзӮ№жҳҜеҸӘйңҖиҰҒеј•е…Ҙйқһеёёе°‘зҡ„и¶…еҸӮж•°пјҢиҖҢдё”еҜ№еҺҹжң¬еӨҡзӣ®ж ҮйҒ—дј з®—жі•зҡ„ж”№еҠЁд№ҹжҜ”иҫғе°ҸгҖӮ

е№¶иЎҢеӨҡзӣ®ж Ү

е№¶иЎҢеҢ–еҚ•зӣ®ж ҮйҒ—дј з®—жі•зҡ„дёҖз§Қж–№жі•жҳҜйҖҡиҝҮеұҖйғЁжҗңзҙўиҝӣиЎҢзҡ„пјҢиҝҷз§Қз®—жі•йҖҡеёёиў«з§°дёәMemeticз®—жі•пјҢдҪҝз”ЁMemeticз®—жі•зҡ„е№¶иЎҢеҢ–жӯҘйӘӨеҰӮдёӢпјҡ
- Step 1: Start with an initial solution \(\boldsymbol{x}\).
- Step 2: Generate a set of neighbor solutions around solution \(\boldsymbol{x}\) using a simple perturbation rule. - Step 3: If the best solution in the neighborhood set is better than \(\boldsymbol{x}\), replace \(\boldsymbol{x}\) with this solution and go to Step 2, else stop.

- еҰӮдҪ•йҖүжӢ©дёҖдёӘи§Јд»ҘеңЁе…¶е‘ЁеӣҙејҖе§ӢеұҖйғЁжҗңзҙў
